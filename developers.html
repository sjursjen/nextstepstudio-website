Developer & Integration Overview

Next Step Studio · KvK 42002265 · jurjen@nextstepstudio.nl

This page is intended for platform partners and API review teams (Garmin, Apple, Strava). It describes how NextPace integrates with third-party platforms, which data is accessed, and how it is used.

About NextPace and Next Step Studio

NextPace is a mobile running training application for Android and iOS, developed by Next Step Studio (registered in the Netherlands, KvK 42002265). The app generates personalised running training plans based on the user's fitness level (VDOT score), goal race distance, target date, and available training days. It includes an AI coaching assistant powered by Anthropic's Claude API.

NextPace is designed for recreational runners — people preparing for a 5K, 10K, half marathon, or marathon who want a science-based, personalised training plan without the cost of a human coach.

Garmin Connect Integration

Next Step Studio is applying for access to the Garmin Connect Developer Program to integrate the Garmin Health API and Activity API into NextPace.

Use case

NextPace uses Garmin activity and health data to automatically track and update the user's training plan. Rather than requiring manual entry of completed workouts, users who own a Garmin device can connect their Garmin account and have their runs, heart rate data, and performance metrics imported automatically.

This data is used exclusively to:

  • Mark scheduled training sessions as completed;
  • Update the user's VDOT score based on recent race and workout performance;
  • Adjust training zones (heart rate, pace) as the user's fitness evolves;
  • Provide the AI coaching assistant with up-to-date context about the user's training load and recovery.

Requested API access

API Data accessed Purpose
Activity API Activity summaries (distance, duration, pace, activity type, start time) Import completed runs and match them to scheduled training sessions in the user's plan

We do not request access to the Health API, Women's Health API, or any biometric/sensor data (heart rate, sleep, stress, pulse oximetry). Our use case is limited to activity tracking.

Data handling

  • Garmin data is stored in Google Firestore, EU region europe-west4 (Netherlands).
  • Data is only accessed with explicit user consent via Garmin's OAuth flow.
  • Users can revoke access at any time from within NextPace or via their Garmin Connect account settings.
  • Garmin data is not shared with any third party other than those listed in our privacy policy.
  • Garmin data is deleted immediately when a user disconnects the integration or deletes their account.
  • We do not use Garmin data for advertising, profiling, or any purpose other than delivering the NextPace training service.

User consent flow

  1. User navigates to Settings → Integrations → Garmin in the NextPace app.
  2. User is redirected to the standard Garmin Connect OAuth consent screen where they can review and approve the requested scopes.
  3. Upon approval, NextPace receives an OAuth token and begins importing activity data.
  4. The user can disconnect at any time via Settings → Integrations → Garmin → Disconnect.

Strava Integration (active)

NextPace already integrates with Strava via the Strava API v3. The integration uses OAuth 2.0 and requests the following scopes:

Scope Purpose
activity:read_all Import completed running activities to match against the training plan
profile:read_all Read athlete profile (name, profile picture) for display in the app

Strava data is handled under the same principles as Garmin data described above.

Apple Health Integration (planned)

A future version of NextPace will support Apple Health integration on iOS. This will allow users to import running workouts and heart rate data directly from the Apple Health store on their device, without requiring a separate account connection.

Requested HealthKit permissions will be limited to:

  • Workout data (running, walking)
  • Heart rate samples
  • VO2max estimates (where available)

All Apple Health data remains on-device unless the user explicitly chooses to sync it to their NextPace account.
